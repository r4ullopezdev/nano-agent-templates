# Contributing

Thanks for contributing to `nano-agent-templates`.

## Workflow

1. Open an issue describing the problem or proposal.
2. Keep changes scoped and explain architectural tradeoffs in the PR.
3. Add or update tests for behavior changes.
4. Run `npm run lint`, `npm run test`, and `npm run typecheck` before opening a PR.

## Standards

- Prefer explicit abstractions over magic helpers.
- Keep examples honest and runnable.
- Mark experimental capabilities clearly.
- Preserve auditability in the execution path.
